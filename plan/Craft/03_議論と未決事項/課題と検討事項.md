## 🔴 ブロッカー級

### 動的 enum × プロセスキャッシュ
- **現象**: FastAPI プロセス内で in-process verify すると、Web 追加した key が enum に反映されない
- **原因**: `scopes/power.py` module import 時に StrEnum と root model が凍結
- **方針**: `build_project(data_root)` factory パターン（[[最終構成]] §10-2 / [[対処方針]] §C.3 と統合）
- **未解決の疑問**: factory 化すると veriq 側の API がどう変わるか？ → 次回 thermal 実装時に実検証

### `@component` の表現力不足
- **datasheet_only**: `SolarCell`、`MaterialEntry` 等が登録できない
- **ネスト config**: `LinkBudget` の `bit_rates.{low,high}` のような階層構造
- **方針候補**（[[対処方針]] §A.5 で `kind` 分岐案を提示済み、Phase 2 で具体化）
  - 拡張オプションで対応 vs. 別デコレータ
  - inner class の再帰処理ルール
